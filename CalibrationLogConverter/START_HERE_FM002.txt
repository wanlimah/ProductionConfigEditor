❓ FREQUENTLY ASKED QUESTIONS
═══════════════════════════════════════════════════════════════════

Q: Is the FM-002 parser ready to use?
A: ✅ YES! It's complete, tested, and production-ready.

Q: Will it work with my existing FM-002 files?
A: ✅ YES! It's designed specifically for FM-002 files.

Q: Do I need to change anything in my workflow?
A: ❌ NO! Just run the application as usual. The FM-002 parser
   automatically handles files with "fm-002" in the name.

Q: What if I have problems?
A: 📖 See FM002_QUICK_START.md → Troubleshooting section
   Run in Debug mode to see detailed diagnostic output.

Q: Can I still use the old BroadcomParser?
A: ✅ YES! It's still there. The FM002Parser is checked first for
   FM-002 files, then falls back to BroadcomParser if needed.

Q: How do I know if FM002Parser is being used?
A: 🔍 Check the Debug output. You'll see messages like:
   "FM002Parser: Processing worksheet 'Logsheet'"

Q: What records get skipped during parsing?
A: The parser skips:
   • Empty rows (completely blank)
   • Rows with "X" markers (error indicators)
   • Rows where BOTH Model AND Serial are empty

Q: Why is my record count lower than expected?
A: This is usually because:
   • Rows have "X" markers (intentionally skipped)
   • Many blank rows at end of sheet (intentionally skipped)
   • This is correct behavior! ✅

Q: How do I get more detailed information?
A: Run the application in Debug mode (F5 in Visual Studio/VS Code)
   and check the Output window for detailed diagnostic messages.

Problem: FM-002 file not loading
Fix: Ensure file is in: C:\Users\wanlimah\Documents\Raw_Data
   OR click Browse button to select manually

───────────────────────────────────────────────────────────────────

Problem: "No suitable parser found"
Fix: Ensure filename contains "fm-002" (case-insensitive)
   Good: FM-002_Field Calibration Report.xlsx
   Good: fm002_report.xlsx
   Bad: field_calibration.xlsx

───────────────────────────────────────────────────────────────────

Problem: No records extracted
Fix: 
   1. Check Debug output for column detection
   2. Verify worksheet name is "Logsheet" or similar
   3. Check that file has Model and Serial columns
   4. See: FM002_QUICK_START.md → Troubleshooting

───────────────────────────────────────────────────────────────────

Problem: Missing due dates
Fix:
   Ensure file has EITHER:
   • A "Due Date" column with dates
   OR
   • "Calibration Date" column AND "Interval" column
