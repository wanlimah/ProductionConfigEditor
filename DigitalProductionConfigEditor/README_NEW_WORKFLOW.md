## 🎓 Basic Usage

### Creating Your First Configuration

#### Step 1: Build Configuration Structure

1. Launch the application
2. You'll see two panels:
   - **Left**: Master Template with all available configurations
   - **Right**: Your New Configuration (initially empty)
3. Browse the Master Template list
4. Click **➕ Add** next to configurations you want
5. Watch them appear in Your New Configuration panel
6. Click **🗑 Delete** to remove any you don't want
7. When ready, click **Next ➡**

#### Step 2: Manage Packages

1. Select a configuration from the dropdown
2. View all packages in that configuration
3. **Add New Package**: Click **➕ Add New Package**
   - Enter package name (e.g., product part number)
   - Add attributes (enable, count, etc.)
   - Click **Create Package**
4. **Edit Package**: Click **Edit** next to a package
   - Modify attributes as needed
   - Changes are applied immediately
5. **Delete Package**: Click **Delete** next to a package
   - Confirm deletion
6. When ready, click **Next ➡**

#### Step 3: Review and Save

1. Review the summary of your configuration
2. Check the XML preview
3. Click **💾 Save**
4. Choose a filename (e.g., `MyProduct_Config.xml`)
5. Select location
6. Click **Save**
7. Done! Your configuration is saved

## ❓ FAQ

### General Questions

**Q: What's the difference between Master XML and New XML?**  
A: Master XML is the read-only template library. New XML is your custom configuration that you're building.

**Q: Can I modify the Master XML?**  
A: No, it's protected and read-only. This ensures it stays clean as a team template.

**Q: Can I create multiple custom XMLs?**  
A: Yes! Create as many as you need - one per product, environment, etc.

**Q: Will my changes affect other users?**  
A: No. Each user creates their own custom XML files. Only the Master template is shared.

---

### Workflow Questions

**Q: Do I have to add all configurations?**  
A: No! Add only the ones you need. That's the power of this new workflow.

**Q: Can I add a configuration multiple times?**  
A: No. Each configuration can only be added once to your XML.

**Q: What happens if I delete a configuration?**  
A: All packages within that configuration are also deleted.

**Q: Can I re-add a deleted configuration?**  
A: Yes! Just click **➕ Add** again from the Master Template.

---

### File Operations Questions

**Q: Where are my custom XML files saved?**  
A: Wherever you choose when clicking Save. Pick any location and filename.

**Q: What's the difference between Save and Save As?**  
A: **Save** overwrites the current file. **Save As** prompts for a new filename.

**Q: Can I open and edit my saved XML files later?**  
A: Yes! Use **📂 Open** to load and continue editing.

**Q: What if I forget to save?**  
A: Changes are not auto-saved. Always click **💾 Save** before closing.

---

### Technical Questions

**Q: What format is the XML file?**  
A: Standard XML (UTF-8 encoded) compatible with your production system.

**Q: Can I edit the XML manually in a text editor?**  
A: Yes, but it's easier and safer to use this editor.

**Q: Is the XML validated?**  
A: Basic validation is performed (required fields, duplicates, etc.)

## 🆘 Troubleshooting

### Application Won't Launch

**Symptoms**: Double-click doesn't start the app

**Solutions**:
1. Ensure .NET 8.0 Runtime is installed
2. Check if `Master_Digital_ProductionUserConfig.xml` exists
3. Run from command line to see error messages

---

### Master Template Not Loading

**Symptoms**: "Master XML not found" error

**Solutions**:
1. Verify `Master_Digital_ProductionUserConfig.xml` is in the same folder as .exe
2. Click **🔄 Reload Master** to retry
3. Check file permissions (should be readable)

---

### Can't Proceed from Step 1

**Symptoms**: "Please add at least one Configuration Node" message

**Solutions**:
1. Click **➕ Add** next to at least one configuration in the Master Template
2. Verify the configuration appears in Your New Configuration panel
3. Then click **Next ➡**

---

### Changes Not Saving

**Symptoms**: Changes disappear after closing

**Solutions**:
1. Always click **💾 Save** before closing
2. Verify the file was saved (check file modification date)
3. Use **💾 Save As** to save to a specific location
